## 1. 개요

### 1.1 제품 정의
미술작가 Jungwhan의 작품을 전시하는 **개인 포트폴리오 웹사이트**.
세련되고 미니멀한 디자인으로 작품 자체에 집중하며, 관리자가 쉽게 작품을 추가/관리할 수 있는 시스템.

### 1.2 핵심 목표
| 목표 | 설명 |
|------|------|
| 작품 전시 | 고해상도 작품을 세련된 갤러리 형태로 전시 |
| 확대 감상 | 작품 디테일을 확인할 수 있는 줌 기능 제공 |
| 쉬운 관리 | 관리자가 코딩 없이 작품 추가/수정/삭제 가능 |
| 세련된 UX | AI가 만든 티가 나지 않는 고급스러운 인터페이스 |
| 다국어 지원 | 한국어/영어 전환으로 국제적 접근성 확보 |

### 1.3 핵심 원칙
- **작품 중심**: UI는 작품을 방해하지 않음
- **미니멀리즘**: 불필요한 요소 배제
- **반응형**: 모바일/태블릿/데스크톱 대응
- **성능**: 이미지 최적화로 빠른 로딩

## 3. 기능 명세

### 3.1 F1: 랜딩 페이지

**설명**: 사이트 첫 진입 화면. 강렬한 첫인상을 주는 히어로 섹션과 대표작 미리보기.

| 항목 | 내용 |
|------|------|
| Input | 페이지 접속 |
| Process | 히어로 이미지 로드, 대표작 썸네일 로드 |
| Output | 풀스크린 히어로 + 대표작 그리드 렌더링 |

**UI 요소**:
- 히어로 섹션: 대표작 1점 풀스크린 배경
- 작가명/타이틀: 미니멀 타이포그래피
- 스크롤 인디케이터: 아래로 스크롤 유도
- 대표작 미리보기: 3~6점 그리드

---

### 3.2 F2: 갤러리 페이지

**설명**: 전체 작품을 그리드 형태로 전시.

| 항목 | 내용 |
|------|------|
| Input | 갤러리 페이지 접속 |
| Process | 작품 목록 조회, 썸네일 렌더링 |
| Output | 작품 그리드 표시 |

**UI 요소**:
- Masonry 또는 균일 그리드 레이아웃
- 호버 시 작품 제목 표시
- 클릭 시 상세 뷰 진입
- Lazy loading 적용

---

### 3.3 F3: 작품 상세 뷰 (모달)

**설명**: 개별 작품을 크게 보고, 확대 감상 가능.

| 항목 | 내용 |
|------|------|
| Input | 작품 클릭 |
| Process | 고해상도 이미지 로드, 메타데이터 표시 |
| Output | 모달 오버레이에 작품 + 정보 렌더링 |

**UI 요소**:
- 작품 이미지 (고해상도)
- 작품 정보: 제목, 연도, 크기, 재료
- 닫기 버튼
- 좌우 화살표 (이전/다음 작품)
- **확대 기능** (핀치줌 또는 클릭줌)

**확대 기능 상세**:
- 데스크톱: 클릭으로 줌 인/아웃, 마우스 드래그로 패닝
- 모바일: 핀치줌, 터치 드래그로 패닝
- 최대 확대 배율: 원본 해상도까지

## 6. 제외 범위 (Out of Scope)

다음 기능은 현재 버전에서 **구현하지 않음**:

- 작품 검색/필터
- 방명록/댓글
- 작품 판매/문의 기능
- 애널리틱스 연동
- 소셜 공유 버튼

> **v2.0에서 구현 완료**: 다국어 지원 ✅, 작가 소개 페이지 ✅, SEO 최적화 (sitemap.ts) ✅

## 7. 기술 스택

| 영역 | 기술 | 이유 |
|------|------|------|
| 프론트엔드 | Next.js 16 (App Router) | SSG/SSR/ISR, 이미지 최적화 내장 |
| 스타일링 | Tailwind CSS 4 | 다크 테마, 반응형 용이 |
| UI 컴포넌트 | Headless UI | 접근성 있는 모달/드롭다운 |
| 이미지 확대 | react-zoom-pan-pinch | 핀치줌/패닝 지원 |
| 백엔드/DB | Supabase (PostgreSQL) | 간편한 설정, 이미지 스토리지 포함 |
| 인증 | bcryptjs + HTTP-only Cookie | 세션 기반 관리자 인증 |
| 배포 | Vercel | Next.js 최적화, 자동 배포 |

## 8. 검증 기준
